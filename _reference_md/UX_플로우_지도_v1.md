# UX 플로우 지도 v1 · 6 플로우 사용자 여정

> **문서 성격** — 우리 서비스의 모든 화면(34 주요 컷 + 17-18 보조 컷)을 *사용자 여정* 결로 정리한 권위 자료. 시간 순서나 카탈로그 번호가 아니라 *사용자가 실제로 만나는 흐름*으로 묶음.
>
> **작성 시점** — 2026.05.19
> **권위 결정** — 마스터 스펙 v4.1 · 무드 가이드 v3.1 · UX 카탈로그 v1 1·2·3부
>
> **사용법** — 구현팀이 *어느 플로우를 작업할지 결정 → 해당 플로우의 컷 목록 확인 → 카탈로그 1·2·3부에서 컷 권위 사양 참조*. 이 문서는 *플로우 단위 작업의 진입점*.

## §2 플로우 1 · 신규 사용자 진입

### 흐름

```
컷 34 v2 (마케팅 랜딩)
    ↓
가입·로그인 화면
    ↓
level_diagnostic_entry (3 질문 진단)
    ↓
컷 1 (진입점 선택 A/B/C)
    ↓
컷 2 (단계 1 인계 — A 진입점의 경우)
```

### 컷 매핑

| 컷 | 자리 | 결의 특징 |
|---|---|---|
| 컷 34 v2 | 마케팅 랜딩 (로그인 전) | 차콜 GNB + 큰 카피 + 세 가치 카드 + 최종 CTA |
| 가입·로그인 화면 | (구현 시 별도 정의) | 이메일·소셜 로그인 등 표준 |
| level_diagnostic_entry | 3 질문 진단 | 디자인씽킹 흐름 + 리서치 + 발산 |
| 컷 1 | 진입점 3 카드 (A/B/C) | A 노랑 강조 + 시작 단계 안내 |
| 컷 2 | 단계 1 인계 | 코치 첫 환영 발화 |

### 결의 결

- **차콜 마케팅 무드 → 흰 작업 무드 전환**. 마케팅 랜딩(컷 34 v2)이 차콜 결, 가입 후 모든 자리는 흰 작업 무드.
- **진단 결과에 따라 초보/숙련 모드 자동 설정** (마스터 스펙 §1.7). 진단 건너뛰기 가능 — 기본값은 초보.
- **A 진입점(처음부터)이 가장 많이 사용**. 컷 1에서 노랑 강조.

### 분기 자리

- B 진입점 선택 → 컷 2 대신 단계 2 진입
- C 진입점 선택 → 컷 2 대신 단계 8 진입 (컷 25 컨셉 시트)
- 진단 건너뛰기 → 초보 모드 기본값

### 마스터 스펙 참조

- §1.4 마케팅 랜딩 페이지
- §1.7 수준 적응 원칙

## §5 플로우 4 · 시스템 영역

### 흐름 — 평행 작동

플로우 4는 *순차 흐름이 아닌 평행 작동*. 어느 단계에서든 GNB에서 진입 가능.

```
[모든 단계에서 GNB 접근 가능]
    ↓
컷 21 (프로젝트 허브) ← 다중 프로젝트 관리
컷 22 (자료실)     ← 5 매크로 그룹 + 산출물 펼침
컷 23 (메모리)     ← 코치 기억 + 사용자 노트
컷 24 (알림)        ← 5 결 + GNB 카운트 배지
```

### 컷 매핑

| 영역 | 컷 | 수준 비교 |
|---|---|---|
| 프로젝트 허브 | 컷 21 | level_compare_hub_beginner/expert |
| 자료실 | 컷 22 | — |
| 메모리 뷰어 | 컷 23 | — |
| 알림 시스템 | 컷 24 | — |

### 결의 결

- **단계 작업과 평행 작동**. 작업 흐름을 방해하지 않음.
- **GNB가 시스템 영역 진입점**. 컷 33 v3(로그인 후 홈)의 GNB 메뉴(자료실·투자·지원)·알림 종에서 접근.
- **자료실은 영구 보존** + source_refs 양방향 추적. 어느 단계에서 만든 자료든 *어디서 왔고 어디로 가는지* 추적.
- **메모리 두 채널** — 코치 기억(자동) + 사용자 노트(수동). 사용자 권한이 두 자리에서 다름.
- **알림 5 결의 GNB 통합 작동** (v4.1) — 종 클릭 시 패널 펼침.

### 분기 자리

- 컷 21 허브에서 *프로젝트 선택* → 해당 프로젝트의 마지막 작업 자리로 진입 (플로우 2 또는 3)
- 컷 22 자료실 산출물 → *원본 단계로 회귀* 진입점
- 컷 24 알림 *마일스톤 도래* → 결과 입력 → 다음 시기 또는 회귀

### 마스터 스펙 참조

- §3 데이터 모델 (자료실·메모리)
- §5.4 알림 5 결 + GNB 통합

## §6 플로우 5 · 재방문 사용자

### 흐름 — 5 영역 진입

```
사용자.return_session.detected (24h+ 만에 재진입)
    ↓
로그인
    ↓
컷 33 v3 (로그인 후 홈) — 5 영역 동시 노출
    ↓
[사용자 선택]
    ↓ 영역 3 클릭     ↓ 영역 4 클릭     ↓ 영역 5 클릭
    프로젝트 계속       새 프로젝트       마감 임박 매칭
    (플로우 2 또는 3)  (플로우 1 일부)   (플로우 6 모듈 4)
```

### 컷 매핑

| 자리 | 컷 |
|---|---|
| 로그인 후 홈 | 컷 33 v3 |
| 진행 중 프로젝트 카드 (영역 3 안) | 마일스톤 도래 / 회귀 권고 / 정상 진행 세 결의 시각 |

### 결의 결

- **로그인 후 홈은 *작업 도구* 정체성** — 마케팅 랜딩(플로우 1)과 정반대 결.
- **흰 GNB 시스템 컨트롤 무드** — 차콜 마감 임박 띠 제거됨 (v4.1).
- **본문 영역 순서가 본인 작업 우선** — 프로젝트 계속 → 새 시작 → 외부 매칭.
- **재방문이 일반적이고 자연스러운 흐름** — 우리 서비스의 사용자가 *14 단계를 한 번에 완주하는 게 아닌 자리* 인정.

### 분기 자리

- 마일스톤 도래 카드 클릭 → 단계 12 또는 13으로 진입 (결과 입력)
- 회귀 권고 카드 클릭 → 권고 단계로 진입
- 정상 진행 카드 클릭 → 마지막 작업 자리로 진입

### 마스터 스펙 참조

- §1.5 로그인 후 홈 페이지

## §8 플로우 간 연결점

플로우 6 결정 셋이 *플로우 간 연결점*으로 작동.

### 연결점 1 · 플로우 1 → 2 진입

```
컷 1 (진입점 선택)
    ↓ A 진입점
컷 2 (단계 1 인계)
    ↓ 진단 결과 적용
플로우 2 영역 2a 시작 (초보 또는 숙련 모드)
```

### 연결점 2 · 플로우 2 단계 8 → 플로우 6 모듈 1

```
컷 14 (우선순위) → 컷 25 (컨셉 시트 4 필드)
    ↓ 4 필드 완성 시
플로우 6 모듈 1 자동 발동 (AI 스토리보드 생성)
```

### 연결점 3 · 플로우 2 단계 8 완성 → 플로우 6 모듈 4 매칭 시작

```
컷 25 컨셉 시트 완성
    ↓
플로우 6 모듈 4 매칭 시작 (사용자 컨셉 기준)
    ↓
플로우 5 로그인 후 홈 영역 5에 맞춤 매칭 노출
```

### 연결점 4 · 플로우 3 단계 13 → 플로우 6 모듈 3

```

    ↓ 발표 모드 진입 CTA

```

### 연결점 5 · 플로우 5 재방문 → 플로우 2 또는 3

```
컷 33 v3 영역 3 (프로젝트 계속 진행)
    ↓ 카드 클릭
플로우 2 또는 3의 마지막 작업 자리로 진입
```

## §10 구현 시 플로우 단위 작업 안내

### 디자이너 작업 단위 추천

| 디자이너 역할 | 추천 플로우 |
|---|---|
| 마케팅 페이지 디자이너 | 플로우 1 |
| 워크북 작업 영역 디자이너 | 플로우 2 |
| 서비스 확장 작업 디자이너 | 플로우 3 |
| 시스템 영역 디자이너 | 플로우 4 + 5 |
| 결과물 우선 영역 디자이너 | 플로우 6 |

### 개발자 작업 단위 추천

| 개발자 역할 | 추천 플로우 |
|---|---|
| 인증·진입 개발자 | 플로우 1 + 5 |
| 단계 작업 흐름 개발자 | 플로우 2 + 3 |
| 시스템 인프라 개발자 | 플로우 4 |
| AI 통합 개발자 | 플로우 6 모듈 1 |
| 외부 데이터 개발자 | 플로우 6 모듈 4 |

### 흐름 매핑

플로우 단위 작업이 자연스러운 이유 — 한 플로우 안의 컷들이 *같은 결의 시각·결정·데이터*를 공유. 한 디자이너/개발자가 *한 플로우를 깊이 작업*하면 결의 일관성이 살아남.

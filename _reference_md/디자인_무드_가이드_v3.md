# 디자인 무드 가이드 v3

> v2에서 도입했던 *세리프 차용*과 *순검정 #0A0A0A*을 모두 제거. **Pretendard 단일 폰트 + 차콜 #2D2D2A 통일**로 무드 시스템 단순화.
>
> v2의 두 무드 시스템(마케팅 / 작업) 골격은 유지. 두 무드의 차이를 *Weight·Size·Letter-spacing·Color 비율*로 만든다.

---

## 0. v3의 정체성 — 한 줄로

*차콜과 노랑으로, Pretendard 한 폰트로. 마케팅은 큰 대비로 부르고, 작업은 미세한 무게로 안내한다.*

---

## 1. 토큰 통일

### 1.1 컬러 토큰

**검정 계열 — 차콜 하나로 통일**:

| 토큰명 | Hex | 역할 |
|---|---|---|
| `--charcoal` | `#2D2D2A` | 모든 짙은 색 자리 |
| `--charcoal-text-on` | `#FFFFFF` | 차콜 배경 위 텍스트 |

순검정 #0A0A0A 사용 안 함.

**노랑·회색 — v1 그대로**:

`--insight-yellow` #F4C430, `--yellow-tint` #FCF3D6, `--yellow-deep` #8B6914, `--yellow-bg` #FFFDF4, `--off-white` #FAFAF7, `--white` #FFFFFF, `--warm-gray` #E5E4DD, `--muted-gray` #6B6A66, `--light-gray` #A09E94, `--bubble-coach` #F8F7F1, `--bubble-user` #F3F2EC, `--marketing-subtle` #F0EFE9

### 1.2 폰트 — Pretendard 단일

**Pretendard Variable** 한 패밀리. 4개 weight만 시스템화.

| Weight | 토큰 | 용도 |
|---|---|---|
| 400 Regular | `--w-regular` | 본문 |
| 500 Medium | `--w-medium` | 강조, 라벨 |
| 600 SemiBold | `--w-semibold` | 단계 헤더, 마케팅 본문 헤드라인 |
| 700 Bold | `--w-bold` | 마케팅 히어로, 큰 숫자 |

세리프·명조 사용 안 함.

---

## 2. 마케팅 무드

**색 비율**: 차콜 60% + 흰 25% + 노랑 10% + 회백 5%. 차콜이 *우세한 무드*.

**타이포**:
- 히어로 헤드라인 56-72px / 700 Bold / -0.5~-1px
- 섹션 헤드라인 32-40px / 600 SemiBold / -0.3px
- 큰 통계 숫자 56-72px / 700 Bold / -1px
- 본문 16px / 400
- 라벨 11px / 500 / +0.4~0.6px / 대문자

**핵심**: 헤드라인 Bold + 타이트 letter-spacing(대담함), 라벨 Medium + 와이드 letter-spacing + 대문자(격조).

**레이아웃**: 차콜 헤더, 대형 사진 히어로, 큰 통계, 인용구 슬라이더, 사례 그리드.

**버튼**: Border-radius 0-4px 사각 (작업 영역 6-8px과 차이).

---

## 3. 작업 무드

**색 비율**: 회백 50% + 흰 30% + 차콜 15% + 노랑 5%. *조용한 무드*.

**타이포**:
- 단계 헤더 22px / 600 SemiBold / -0.3px (v3 새 추가)
- 슬롯 헤더 16px / 500 Medium
- 카드 제목 13px / 500 Medium
- 본문 12-13px / 400
- 메타 라벨 11px / 400
- 작은 라벨 10-11px / 500 / +0.3px

**핵심**: Bold 사용 안 함. SemiBold가 최대. *목소리를 낮추는 무드*.

**v1 그대로 유지**: 노랑 사용 규칙, 분할 화면 패턴, 카드 상태, 코치 모드 6종, 8px grid, Border Radius 6-16px, 시맨틱 매핑, 출처 추적 칩 파란 톤 예외.

**v3 새 추가**:
- 단계 헤더 SemiBold 22px
- 완료 축하 화면 = 차콜 인터스티셜 (Pretendard SemiBold)
- 워크북 인용 강조 = Pretendard 이탤릭 + 따옴표 + 노랑 카드 (세리프 안 씀)

---

## 4. 두 무드 차이 한 표

| | 마케팅 | 작업 |
|---|---|---|
| 색 비율 | 차콜 60% | 회백 50% |
| Bold | 헤드라인에 사용 | 사용 안 함 |
| 최대 Size | 72px | 22px |
| Letter-spacing | 타이트/와이드 대비 강함 | 거의 자연 |
| Border-radius | 0-4px 사각 | 6-16px 둥근 |
| 사진 | 대형 우선 | 거의 없음 |
| 노랑 위치 | CTA, 키워드 | 통찰의 한 자리 |
| 노랑 양 | 10% | 5% |

같은 폰트 + 같은 차콜 + 같은 노랑으로 *완전히 다른 두 무드*. 조화와 대비가 한 시스템 안에서 작동.

---

## 5. 전환 자리

차콜 인터스티셜 2-3초. 차콜 배경 + 작은 노랑 점 + Pretendard SemiBold 짧은 메시지.

- 슬롯 완료: 0.8초 토스트
- 단계 완료: 2-3초 인터스티셜
- 프로젝트 완료: 5초 차콜 화면

---

## 6. 15초 핵심 원칙

1. 어느 영역인가 — 마케팅 vs 작업
2. 노랑은 통찰의 순간에만
3. Pretendard 한 폰트로 다 한다 — Weight·Size·Letter-spacing의 미세 조정
4. 모든 짙은 색은 차콜
5. 마케팅 = 큰 대비 + 사진 / 작업 = 작은 무게 + 카드

## 한국어 타이포 사양 (v3.1 신규)

한국어 서비스의 가독성 기본은 어절 보존이다. 단어 중간 줄바꿈으로 의미 전달이 끊기지 않게 한다.

### 전역 적용

```css
* {
  word-break: keep-all;
  overflow-wrap: break-word;
}
```

### 작동
- 한국어: 어절 단위 줄바꿈 (단어 보존)
- 영어: 단어 단위 줄바꿈 (정상 동작)
- 너무 긴 단어/URL: 강제 줄바꿈 (컨테이너 넘침 방지)

### 적용 범위
전역. 마스터 스펙 §1.6 한국어 타이포 사양과 동일 결정. 예외 없음.

## §1 헌법 · 우리 서비스의 결

### §1.1 미션

> 예비 창업자가 *아이디어 → 사업화 → 투자 연결*까지 도달하는 코칭 도구.

워크북 14 단계를 한 바퀴 돌면 사용자는 *피치 덱을 들고 투자자 만남* 자리에 도달한다. 우리는 *디자인씽킹 교육 도구*가 아니라 *사용자가 외부 세계와 만나는 자리에 데려가는 동행자*다.

세 가지 차별 자리 —
1. *디자인씽킹 워크북 정본*에 기반한 코칭 (HiDD 워크북 정신)
2. *사용자 수준에 적응*하는 코칭 강도 (초보 ↔ 숙련)
3. *결과물의 외부 통용성* (피치 덱·시제품·지원사업 매칭이 바로 외부로 갈 수 있는 질)

### §1.2 6 원칙

워크북 정신에서 도출된 5 원칙에 *수준 적응 원칙 1종*이 추가되어 v4 헌법은 6 원칙이다.

**원칙 1 · 워크북 정본 따른다.** HiDD 디자인씽킹 워크북·HCI 리서치 워크북의 모든 메서드·정신·용어가 *서비스 안에서 그대로 작동*. 외부 자료나 코치 임의 판단이 워크북을 *대체하지 않는다*.

**원칙 2 · 사용자 발화·행동이 자료의 출처.** 사용자가 *말한 것*과 *행동에서 보인 것*이 작업의 원자 자료. AI나 코치가 *추측한 것*은 가설로 명시되고 사용자 검증 필요.

**원칙 3 · 가설은 검증 전엔 가설.** 어떤 자리도 *검증되지 않은 채로 결론으로 굳지 않는다*. 단계 5 잠재 니즈도, 단계 11 사업 타당성도, 단계 14 매칭 추천도 *모두 가설 상태*에서 시작.

**원칙 4 · 작업의 결은 자료실에 영구 보존.** 모든 산출물·버전·작성자·시간·source_refs가 *자동 누적*. 사용자가 삭제 명시할 때까지 보존.

**원칙 5 · 회귀는 패배가 아니다.** 단계 10 검증 실패가 단계 4로의 회귀를 트리거하는 게 *자연스러운 흐름*. *직선이 아니라 나선*이 워크북 정신.

**원칙 6 · 수준에 따라 강도 조절.** 같은 워크북 정본을 *초보 사용자는 함께 손잡고, 숙련 사용자는 점검 톤으로* 적용. 결과물의 질은 두 수준에서 동일, 코칭 강도만 갈라진다. (§1.7 상세)

### §1.3 워크북 정신 7가지

워크북에서 추출된 *코치 발화의 결*을 만드는 7 정신.

1. *How보다 Why* · 표면 답이 아니라 깊은 동기로 내려간다
2. *결론이 아니라 가설* · 모든 발견은 다음 단계의 입력
3. *시각화로 사고를 외부화* · 글로만 보지 않는다
4. *극단에서 보편을 본다* · Extreme User가 잠재 니즈를 가장 잘 드러낸다
5. *직선이 아니라 나선* · 회귀가 패배가 아닌 도구
6. *입장 정리가 결정* · 모든 렌즈가 통과일 필요 없음. 약한 자리를 *아는 것*이 중요
7. *외부 언어로 옮긴다* · 발견한 결을 투자자·파트너의 언어로 변환하는 게 도착점

### §1.3.1 코치 정체성

코치 표시 이름은 **Kevin**. UI 헤더·입력 placeholder·마케팅 시안·코치 발화 카피는 이 이름을 쓴다. (구 시안의 리아/Lia 명칭은 폐기.)

코치는 *답을 알려주지 않는다*. *좋은 질문을 던지고, 함께 정리한다*. 사용자가 *통찰에 도달하는 자리*를 *코치가 동반*한다.

런타임 발화 모드는 §2.2 *6종* (환영 · 듣는 중 · 짚어주는 중 · 제안 중 · 분기 발동 · 학습 회고). v3 이후 *격려 중*·*대기 중* 등 레거시 라벨은 위 6종에 흡수해 기술한다.

v3 이후 유료 *다른 코치 페르소나* 모듈이 있어도 MVP 기본 코치는 Kevin이다. 페르소나가 바뀌어도 §1.2 6원칙·§1.3 워크북 정신은 동일하고 *말투·비유*만 달라진다.

### §1.4 마케팅 랜딩 페이지 (로그인 전)

> 처음 만나는 사용자가 우리 서비스를 받아들이는 첫 무대. 가치 제안 + 회원가입·로그인 유도. 마케팅 무드가 페이지 전체를 지배한다.

**정체성** — 사용자 컨버전 도구. 우리 서비스가 무엇인지 첫 시야에서 받게 한다.

**구조 4 영역 (수직 흐름)**

영역 1 · 차콜 GNB (마케팅 무드)
- 좌측: 로고(노랑 사각 + 차콜 아이콘) + 서비스 이름
- 중앙 메뉴: 소개 · 우리가 하는 일 · 사례 · 요금 · 문의
- 우측: 로그인(텍스트) + 시작하기(노랑 CTA 버튼)

영역 2 · 히어로 (2 컬럼)
- 좌측 컬럼
  - 워크북 권위 라벨 (HiDD 디자인씽킹 워크북 기반)
  - 큰 헤드라인 (44px Bold · letter-spacing -1px · 3 줄 · 마지막 줄 노랑 #8B6914)
  - 부제 (15px · 우리 서비스 한 문장 정의)
  - CTA 2종 (무료로 시작하기 차콜 · 서비스 데모 보기 흰)
  - 소셜 증명 (4 사용자 아바타 + 현재 N팀 작업 중 카피)
- 우측 컬럼
  - 실제 코칭 화면 미리보기 박스 (단계 5 코치 발화 + 디자인씽킹 정신 인용)
  - 좌상단 라벨 (실제 코칭 화면)
  - 우측 회전 노랑 카드 (마감 임박 프로그램 · 자금 큰 숫자 · D-N)

영역 3 · 세 가치 카드 (3 컬럼)
- 상단 라벨 (왜 서비스명인가) + 큰 헤드라인 + 부제
- 카드 1 (코치와 함께): 회백 배경 + 차콜 아이콘 박스 + 노랑 아이콘
- 카드 2 (AI 시각 생성): 노랑 배경 + 노랑 외곽 + 우상단 우리 차별 라벨 (중앙 강조)
- 카드 3 (외부 도착점): 회백 배경 + 차콜 아이콘 박스 + 노랑 아이콘
- 각 카드 하단: 키워드 라벨 + 화살표 진입점

영역 4 · 최종 CTA (가운데 정렬)
- 상단 라벨 (시작하세요)
- 큰 헤드라인 (32px Bold)
- 안내 카피 (3 질문 진단 + 14 단계 흐름 + 단계 8부터 매칭 시작)
- CTA 2종 (무료로 시작하기 차콜 · 로그인 흰)
- 마이크로카피 (신용카드 등록 불필요 · 첫 프로젝트 무료 · 30초 안에 시작)

**금지된 자리**
- 차콜 마감 임박 띠 (히어로 우측 회전 노랑 카드가 같은 신호를 이미 가짐 · 중복 제거)
- 본문 마감 임박 그리드 (마케팅 랜딩은 지원사업 검색 서비스가 아님)

**시각 무드 결**
- 영역 1: 마케팅 무드 차콜 헤더
- 영역 2: 큰 카피 + 노랑 강조 + 회백 배경
- 영역 3: 흰 배경 + 가운데 카드만 노랑 (마케팅 강조 결)
- 영역 4: 회백 배경 + 가운데 정렬 (행동 결정 자리)

### §1.5 로그인 후 홈 페이지

> 본인의 자리를 관리하는 시스템 컨트롤 바 + 작업 영역. 사용자 상태(첫 방문 / 재방문)에 따라 일부 영역이 조건적 노출.

**정체성** — 사용자 작업 도구. 본인 프로젝트와 마감 임박이 한 자리에서 작동.

**구조 5 영역 (수직 흐름)**

영역 1 · 흰 GNB (시스템 컨트롤 바)
- 좌측: 로고(차콜 사각 + 노랑 아이콘) + 서비스 이름
- 중앙 메뉴: 홈(활성 차콜) · 프로젝트 · 자료실 · 투자·지원
- 우측: 검색바(⌘ K 단축키) · 알림 종(노랑 카운트 배지) · 사용자 썸네일 드롭다운(노랑 아바타 + 이름)

영역 2 · 히어로 (좌측 정렬)
- 작은 라벨 (DESIGN THINKING COACH · 홈)
- 짧은 인사 카피 (26px Bold · 사용자 이름 호명)
- 부제 한 줄 (진행 프로젝트 N개 + 마일스톤 도래 + 마감 임박 N개)

영역 3 · 프로젝트 계속 진행 (재방문 시 활성, 첫 방문은 빈 상태 안내)
- 헤더: 북마크 아이콘 + 라벨 + 전체 보기 진입점
- 프로젝트 카드 2열 그리드
- 카드별 결의 시각 위계
  - 마일스톤 도래: 노랑 카드 + 노랑 외곽 + 우상단 라벨 + 코치 안내 노랑 박스
  - 회귀 권고: 흰 카드 + 회백 라벨 + 코치 회귀 권고 회백 박스
  - 정상 진행: 흰 카드 + 회백 메타 정보
- 카드 안 14 점 진행 막대 (전체 카탈로그 진행 인디케이터 패턴 일관)
- 액션: 이어서 진행 (차콜) / 자료실 (흰)

영역 4 · 새 프로젝트 시작 · 어디서 출발하세요?
- 헤더: 라우트 아이콘 + 라벨 + 부제 (새 아이디어를 코치와 함께 정리)
- 진입점 3 카드
  - A · 처음부터 차근차근: 노랑 카드 + 가장 많이 라벨 (14 단계 전체)
  - B · 아이디어 가진 채로: 흰 카드 (단계 2부터)
  - C · 프로토타입 빨리: 흰 카드 (단계 8부터)
- 영역 하단 안내: 진입 후 3 질문 진단으로 코칭 강도 자동 설정 (§1.6 발동)

영역 5 · 지금 신청 가능한 프로그램 · 마감 임박
- 옅은 노랑 배경 #FFFDF4 + 노랑 외곽으로 외부 세계와의 다리 결 명시
- 헤더: 불꽃 아이콘 + 라벨 + 부제 + 필터 탭 (전체·정부·지자체·VC·민간)
- 4 카드 그리드 (가장 임박한 자리부터)
- 카드: 기관 라벨 + 프로그램 이름 + 짧은 설명 + 자금 큰 숫자 + D-N 칩
- 첫 카드만 노랑 외곽 + 노랑 D-N 칩 (가장 임박)
- 하단 데이터 출처 박스 (K-Startup·창업진흥원·중기부·서울창업허브·The VC·DealSite·17개 지자체 큐레이션·매일 자정 업데이트)

**시각 무드 결**
- 영역 1: 흰 시스템 컨트롤 무드 (마케팅 랜딩 차콜 GNB와 정반대)
- 영역 2·3·4: 흰 작업 무드 (서비스 내부 결)
- 영역 5: 옅은 노랑 마케팅 무드 (외부 연결 결만 분리)

**금지된 자리**
- 상단 차콜 마감 임박 띠 (GNB 알림 카운트 배지가 같은 신호를 더 가벼운 결로 가짐)
- 마감 임박이 GNB 위로 들어오지 않음 (GNB는 사용자 컨트롤 자리이지 콘텐츠 자리가 아님)

**두 페이지의 결 차이 (§1.4 vs §1.5)**

| 자리 | 마케팅 랜딩 §1.4 | 로그인 후 홈 §1.5 |
|---|---|---|
| 정체성 | 컨버전 도구 | 작업 도구 |
| GNB 배경 | 차콜 (마케팅 무드) | 흰 (시스템 컨트롤) |
| GNB 메뉴 | 소개·우리가 하는 일·사례·요금·문의 | 홈·프로젝트·자료실·투자·지원 |
| GNB 우측 | 로그인 + 시작하기 노랑 CTA | 검색 + 알림 종(카운트) + 사용자 썸네일 |
| 히어로 카피 | 44px 큰 카피 + 워크북 권위 라벨 | 26px 짧은 인사 카피 |
| 마감 임박 위치 | 히어로 우측 회전 노랑 카드 1개 | GNB 알림 카운트 + 본문 영역 5 |
| 최종 CTA | 있음 (무료로 시작하기) | 없음 (이미 로그인 상태) |

### §1.6 한국어 타이포 사양 (v4.1 신규)

> 한국어 서비스의 가독성 기본은 어절 보존이다. 단어가 줄바꿈으로 잘리면 의미 전달이 끊긴다.

**전역 적용 사양**
```css
* { word-break: keep-all; overflow-wrap: break-word; }
```

**작동**
- 한국어: 어절 단위로 줄바꿈 (단어 보존)
- 영어: 단어 단위로 줄바꿈 (정상 동작)
- 너무 긴 단어/URL: 강제 줄바꿈 (컨테이너 넘침 방지)

**적용 범위** — 전역. 마스터 스펙 · 무드 가이드 · 카탈로그 모든 컷에서 동일 적용. 예외 없음.

### §1.7 수준 적응 원칙

> 같은 워크북 정본을 사용자 수준에 따라 *다른 강도*로 전달한다. 결과물의 질은 동일, 코칭 강도만 갈라진다.

**두 수준 MVP** (글로벌 수준)
- 초보 모드 · 디자인씽킹·메서드 처음 접하는 사용자
- 숙련 모드 · 워크숍 경험 있거나 디자인씽킹 익숙한 사용자
- v2 확장: 길 C(단계별 수준) — 사용자가 *전체 수준 + 각 단계마다 다른 수준*

**측정 방법 두 가지 결합**
- 명시적 진단: 진입점 선택 직후 *3 질문* (디자인씽킹 흐름·리서치 메서드·발산 메서드)
- 행동 신호: 코치 안내 건너뛰기, 직접 적기 비율, 메서드 호출 빈도, 용어 칩 클릭 빈도

**8 자리에서 갈라지는 두 모드**
1. *단계 헤더 부제 길이* — 초보 풀어 설명 / 숙련 키워드만
2. *워크북 정신 인용 박스 등장 여부* — 초보 자주 / 숙련 분기 발동 시만
3. *코치 발화 문단 수* — 초보 3-4 문단 / 숙련 1-2 문단
4. *코치 제안 후보 등장 여부* — 초보 펼침 / 숙련 없음 + 직접 적기
5. *가이드 질문·예시·source 명시 여부* — 초보 카드 내부 명시 / 숙련 호출 시 등장
6. *시각 자료 등장 여부* — 초보 세 원 다이어그램 자동 / 숙련 헤더 옆 작은 보기 버튼
7. *회귀 권고 카드의 풀어 설명 vs 한 줄 트리거* — 초보 이유 풀이 / 숙련 트리거만
8. *시스템 안내 박스 등장 여부* — 초보 자주 / 숙련 분기 신호 사전 안내만

**막힘 감지 신호 3종**
- *시간 신호* · 한 카드 5분+ 진전 없음 (단계마다 적정 시간 다름)
- *명시적 신호* · *막혔어요·모르겠어요·예시 보여주세요* 발화 또는 *도움 받기* 버튼
- *비활동 신호* · 탭 전환·30분+ 입력 없음

**개입 발동**
- 막힘 감지 시 *그 단계에 한해 잠시 초보 모드 전환*
- 코치가 *제안 후보 풍부히 펼침 + 메서드 함께 수행 모드 제안*
- 사용자가 막힘 풀면 *자동으로 원래 수준 복귀*

**학습 경로**
- 단계 완료 시 *학습 회고 모듈* 자동 트리거 (`stage.completed` 이벤트)
- 점진적 자율 확대 — 같은 단계 안에서도 처음 카드는 코치 제안 풍부, 마지막 카드는 직접 적기 기본
- 이전 프로젝트 학습 인식 — 두 번째 프로젝트 시작 시 *이미 익숙한 단계는 자동 숙련 모드*

**결과물 우선 모듈은 예외** (§7.3 참조)
- 피치 덱 빌더·발표 모드·AI 시각 생성·외부 데이터 통합은 수준 적응 따르지 않음
- 결과물의 질이 사용자 수준과 무관하게 *전문 수준*으로 보장
- 코치 역할은 *진입·해석 도움*만, 결과물 자체에 영향 안 줌

## §2 런타임 · 코치의 작동 골격

### §2.1 슬롯 상태 모델

각 단계는 *슬롯들*로 구성되고, 각 슬롯은 다섯 상태 중 하나:
- `empty` — 시작 전
- `in_progress` — 사용자·코치 함께 작업 중
- `complete` — 사용자가 *충분*이라고 결정
- `revisit` — 회귀 권고로 다시 열림
- `frozen` — 다음 단계로 진행됐으나 잠금되지 않은 상태

### §2.2 코치 모드 6종

코치의 발화 결은 *6 모드* 중 하나로 작동:
1. *환영 인사* — 진입/재진입 시 인사
2. *듣는 중* — 사용자 입력 기다림, 발화 최소
3. *짚어주는 중* — 사용자 작업 점검, 부드러운 안내
4. *제안 중* — 코치 후보 제시, 사용자 결정 대기
5. *분기 발동* — 워크북 정신 위반 감지 시 적극 개입
6. *학습 회고* — 단계 완료 시 *방금 한 것의 의미* 정리

수준 적응이 *각 모드의 발화 양·톤*에 영향. 모드 자체는 두 수준 동일.

### §2.3 런타임 사용자 상태

```
user_state {
  user_id
  global_level: 'beginner' | 'expert'  // 진단 + 행동 신호로 결정
  stage_levels?: { [stage_id]: level }  // v2 길 C 확장 시
  current_session_intervention?: stage_id  // 막힘 감지로 임시 초보 모드 발동 중인 단계
  return_session: boolean  // 24h+ 만에 재진입
  active_project_id
  active_stage_id
  active_slot_id
}
```

### §2.4 분기 시스템

워크북 정신 위반 감지 시 *분기 발동*. 분기 자체는 두 수준 동일하게 발동, 발화 양만 수준에 따라 조절.

분기 카탈로그는 §9 참조.

## §5 이벤트 모델

### §5.1 슬롯·단계 이벤트

- `slot.entered(slot_id, user_id, timestamp)`
- `slot.updated(slot_id, item_id, action)`
- `slot.completed(slot_id, user_id)`
- `stage.completed(stage_id, user_id)` → 학습 회고 모듈 트리거

### §5.2 사용자 상태 이벤트 (v4 신규)

- `user.first_session` → 로그인 후 홈(§1.5) 첫 방문 모드 (프로젝트 계속 진행 영역 빈 상태 안내)
- `user.return_session.detected` (24h+ 지난 재진입) → 로그인 후 홈 재방문 모드 + *프로젝트 계속 진행* 영역 활성
- `level.changed(from, to, trigger)` → 수준 적응 발동
- `stuck.detected(card_id, signal_type)` → 막힘 감지
- `intervention.activated(stage_id, reason)` → 임시 초보 모드 전환
- `intervention.deactivated(stage_id)` → 원래 수준 복귀
- `learning_review.delivered(stage_id, content)` → 학습 회고 전달

### §5.3 회귀 이벤트군

- `regression.suggested(from_stage, to_stage, reason)` — 코치 권고
- `regression.accepted` / `regression.declined`
- 회귀 이력은 artifact의 versions_history에 영구 보존

### §5.4 알림 5 결 (v4 확장: 4 → 5)

각 결마다 시각 톤·기본 액션·트리거 신호가 다름.

1. *마일스톤 도래* — 노랑 카드 + 결과 입력 액션
2. *회귀 권고* — 흰 카드 + 노랑 외곽 + 회귀/계속 액션 (사용자 동의 필요)
3. *협업* — 흰 카드 + 회백 (정보 알림, 액션 없음)
4. *학습 회고* — 흰 카드 + 회백 (코치 회고, 액션 없음)
5. *지원사업 마감 임박* (v4 신규) — 노랑 카드 + 신청 준비 액션
   - 트리거: 9일 전·3일 전·당일 아침 3 단계
   - 사용자가 *신청 준비 중* 표시한 프로그램만 알림

**GNB 통합 작동 (v4.1 신규)**

로그인 후 홈(§1.5)의 GNB 우측 알림 종 아이콘에서 5 결이 통합 표시.

- 알림 종 우상단 노랑 카운트 배지(미확인 알림 개수)
- 종 클릭 시 알림 패널 펼침 — 5 결이 시간순 정렬
- 결별 시각 톤은 카탈로그 §컷 24 결정 유지
- 알림 패널은 GNB 자리에서만 등장 (본문 영역 침범 안 함)

GNB 알림 종이 *시간 압박 + 사용자 작업 신호 + 외부 매칭 신호*를 한 자리에 모은다. 이전 v4의 차콜 마감 임박 띠가 가졌던 역할이 더 가벼운 결로 GNB에 흡수됨.

### §13.3 용어 칩 UI 패턴

**칩 위치** · 카드 안 작은 정보 아이콘 옆 *용어* 라벨

**칩 시각 강도** (수준 적응과 결합)
- 초보 모드: 보통 시인성 (회색 외곽 + 회색 i 아이콘)
- 숙련 모드: 옅은 시인성 (점선 외곽 + 옅은 회색, 호버 시 강조)
- 두 모드 모두 *항상 사용자 의도 시 클릭 가능*

**클릭 시 펼침 (모달 아닌 카드 안 인라인)**
1. 학술 용어 (예: Tacit Need) + 그룹 라벨 (예: *디자인씽킹 워크북 용어*)
2. 짧은 설명
3. 우리 서비스 한국어 대응
4. 이 단계에서 어떻게 발견하나
5. 더 자세히 · 워크북 / 관련 용어 더 보기

**닫기** · 용어 칩 위치에 *용어 닫기* 칩으로 변형, 노랑 강조
